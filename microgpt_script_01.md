# microgpt 系列 第 1 集 字幕稿

[Image 01]
大家好，今天我们用 200 行 Python
看懂语言模型到底是怎么训练出来的
主角是 Karpathy 写的 microgpt.py
它没用 PyTorch，也没用 NumPy
甚至没有用任何数组库
只用了 os math random 三个 Python 标准库
整个文件没有依赖外部数学包
但它的训练流程完整可运行
从数据准备，到训练，再到推理生成
读懂这 200 行，就能抓住 GPT-4 训练的核心
两者最大的差别，主要在模型规模
这个系列分三集，今天是第一集
我们讲数据和 tokenizer
也讲模型训练前的参数骨架
也就是 LLM 开始训练前，要先搭好的舞台
如果你用过 ChatGPT，你知道它能对话
但模型本身做的事，其实简单得多
它只在做一件事

[Image 02]
给定一段文字，猜下一个字是什么
就这么简单
比如输入“床前明月”
模型会输出一组概率
它认为下一个字最可能是“光”
也可能是“色”，或者是“夜”
然后我们把“光”接上去
让模型继续预测下一个字
这件事看起来很小
但只要预测得足够准
把每一步结果接起来
模型就能写诗、写代码，也能回答问题
所有 LLM 本质上都在做这件事
microgpt 也是一样
它不是在理解名字的含义
而是在学习一个名字里
下一个字符通常是什么
理解了这一点
后面的内容都是围绕它展开的
要学会预测下一个字符
先要给模型看大量真实样本

[Image 03]
microgpt 使用的数据，是一份英文人名清单
里面有三万多个名字
从 emma olivia liam 这些常见名字
到一些很少见的名字
它们就是模型要模仿的对象
代码写得非常朴素
按行读取文件，去掉每行多余空白
再存成一个列表，然后把顺序打乱
打乱顺序很重要
如果按字母顺序训练
模型一开始只看 a 开头
后面才看到 z 开头
学到的数据分布就会偏
打乱之后
每一步训练看到的数据，才更接近随机分布
真正的 GPT 训练，数据规模大得多
几乎是互联网级别的文本
但本质一样，都是把一堆字符串随机喂给模型
数据量的差距，最终会决定能力差距
但它们的数据形态是一致的
模型只能处理数字，不能直接处理字符
所以需要一个翻译层
这就是 tokenizer，把字符映射成整数

[Image 04]
microgpt 的 tokenizer 非常简单
先收集所有出现过的字符
也就是 26 个英文字母
然后按字母顺序排好
每个字母对应一个 id
a 对应 0，b 对应 1，c 对应 2
一直到 z 对应 25
这就是字符级 tokenizer
GPT-4 用的不是字符级
而是子词级，也就是 BPE
它会把 hello 这类常见组合压成一个 token
但核心原理没变
都是先建立一张查找表
字符串进来，整数序列出去

[Image 05]
只有这张表还不够
模型还要知道，一个名字从哪里开始
又在哪里结束
所以额外加一个特殊符号，叫做 BOS
全称是 Beginning of Sequence
它的 id 是 26，正好接在最后一个字母后面
每个名字训练时
前后都会夹上 BOS
比如 emma 会变成
BOS e m m a BOS
开头的 BOS 表示开始
结尾的 BOS 表示说完了
模型要学会预测它
等到推理生成时
这个结束信号非常重要
模型靠它判断何时停下
讲完数据之后，再看模型本体
模型本体说白了，就是一堆数字
这些数字叫参数
它们一开始是随机的
由高斯分布采样得到
训练要做的事，就是不断调整这些数字
让模型从随机变聪明

[Image 06]
microgpt 把所有参数都放在一个字典里
这个字典叫 state_dict
我们看里面有什么
wte 是 token 嵌入表
把每个字符 id 变成 16 维向量
wpe 是位置嵌入表
告诉模型字符所在的位置
接着是一层 Transformer，里面有六个矩阵
注意力用 wq wk wv wo
前馈网络用 fc1 和 fc2
最后是 lm_head
把模型最终输出映射回 27 个字符的概率
这里每一项都是矩阵
矩阵里面装的
都是 microgpt 自己写的 Value 类对象
下集我们会重点讲它

[Image 07]
所有参数加起来
这个模型一共有多少个数字
打印结果是四千二百多个
这就是 microgpt 这个 LLM 的全部知识
它会写出什么名字
它学到的字符规律
都编码在这些浮点数里
GPT-4 估计有上万亿个数字
听起来差距巨大
但每个数字的角色，和这里完全一样
都是一个浮点数，等待被梯度下降调整
参数越多
模型能记住的模式越复杂
但训练原理不变
到这里，模型骨架就搭好了
我们有了数据，有了 tokenizer
也有一个装着随机数字的模型
现在的问题是
这些数字如何组合起来
让一个字符输入进去
变成下一个字符的预测
下一集
我们让数据流过模型
讲清楚 Transformer 的核心机制
也就是注意力
