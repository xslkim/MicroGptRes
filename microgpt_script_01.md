# microgpt 系列 第 1 集 字幕稿

[Image 01]
大家好 今天我们用 200 行 Python
理解一个语言模型是怎么训练出来的
我们要拆解的文件叫 **microgpt.py**
是 Karpathy 写的
它没有用 PyTorch 没有用 NumPy
甚至没有用任何数组库
只用了三个标准库 os math 和 random
整个文件里没有一行
依赖外部数学包的代码
但它是一个完整可运行的训练流程
从准备数据 到训练 到推理生成
读懂这 200 行
你就读懂了 GPT-4 训练的**核心原理**
它们之间的差别 主要是规模
这个系列分三集 今天是第一集
我们讲数据 讲 **tokenizer**
讲模型的参数骨架
也就是 **LLM** 训练之前要先搭好的舞台
如果你用过 ChatGPT
你知道它在做对话
但模型本身做的事 比这个简单得多
它只在做一件事
[Image 02]
**给定一段文字 猜下一个字是什么**
仅此而已
我们把床前明月喂给模型
它输出一个概率分布
告诉你下一个字最可能是光
其次可能是色 再次可能是夜
然后我们把光接上去 让它再猜下一个字
这件事看上去微不足道
但只要它猜得足够准
把每一步预测拼起来
模型就能写诗 写代码 回答问题
所有 LLM 在做的
根本上都是这一件事
microgpt 也是一样
它不是在理解名字
它是在学 一个名字里
下一个字符通常是什么
理解了这一点
后面所有内容都是为这件事服务的
要学下一个字符是什么
你需要给模型看大量真实样本
[Image 03]
microgpt 用的数据
是一份英文人名清单
三万多个名字
从 emma olivia liam 这些常见名字
到一些非常罕见的名字
这就是它要模仿的对象
代码里这一段非常朴素
把文件按行读进来 去掉空白
存成一个列表 然后**打乱顺序**
打乱顺序很重要
如果按字母顺序训练
模型前期只看到 a 开头
后期只看到 z 开头 它会学偏
打乱让每一步训练接触的数据
是随机分布的
真实的 GPT 训练用的数据规模大得多
是整个互联网级别的文本
但本质和这里一样
一堆字符串 随机喂给模型
数据量上的差距
最终决定了模型能力的差距
但它们的形态是一致的
模型只能处理数字 不能处理字符
所以我们需要一个翻译层
叫 **tokenizer** 把字符映射成整数
[Image 04]
microgpt 的 tokenizer 极其简洁
把所有名字里出现过的字符
也就是 26 个英文字母排好序
每个字母对应一个 id
a 是 0 b 是 1 c 是 2
依此类推 z 是 25
这就是字符级 tokenizer
GPT-4 用的不是字符级
而是**子词级** 也叫 **BPE**
把 hello 这种常见组合压成一个 token
但原理没变 建一张查找表
字符串进来 整数序列出去
[Image 05]
光这样还不够
模型怎么知道一个名字开始了
又怎么知道一个名字结束了
我们额外加一个特殊符号叫 **BOS**
全称是 Beginning of Sequence
它的 id 是 26
紧接在最后一个字母后面
每个名字训练的时候
前后各夹一个 BOS
比如 emma 会被处理成
BOS e m m a BOS
开头的 BOS 告诉模型 开始了
结尾的 BOS
是它要学着预测出来的 我说完了的信号
后面看推理的时候会知道
这个收尾信号至关重要
模型靠它判断什么时候停下来
讲完数据 讲模型本体
模型本体说白了
就是**一堆数字** 称为参数
这些数字一开始是随机的
具体是用一个高斯分布采样出来的
训练就是不断调整
让模型从随机变得聪明
[Image 06]
microgpt 把所有参数存在一个字典里
叫 state_dict
我们看下里面有什么
**wte** 是 token 嵌入表
把每个字符 id 变成 16 维向量
**wpe** 是位置嵌入表
告诉模型这个字符在第几个位置
接着是一层 Transformer 里的六个矩阵
注意力的 wq wk wv wo
前馈网络的 fc1 和 fc2
最后是 **lm_head**
把模型最终的输出
映射回 27 个字符的概率
每一项都是一个矩阵
里面装的全是
microgpt 自己写的**Value 类**对象
下集我们会重点讲它
[Image 07]
所有参数加起来
这个模型一共多少数字
打印出来是 4200 多个
这就是 microgpt 这个 LLM **全部的知识**
它会写什么样的名字
它学到的字符规律
全部编码在这 4200 个浮点数里
GPT-4 估计有**上万亿个数字**
听起来天差地别
但每一个数字的角色和这里一模一样
都是一个浮点数 等待被梯度下降调整
参数越多 模型能记住的模式就越复杂
但训练原理不变
到这里 骨架就搭好了
我们有了数据 有了 tokenizer
有了一个装着 4200 个随机数字的模型
现在的问题是
这些数字怎么被组合起来
让一个字符进去
变成下一个字符的预测
下一集 我们让数据流过模型
讲清楚 Transformer 的**核心机制注意力**
