> 本系列基于 Andrej Karpathy 的 microgpt.py 拆解 LLM 训练原理。共三集。

上一集我们算出了 loss——一个表示"模型有多差"的数字。

这一集，我们让这个数字反过来，去修正所有参数。这是训练真正发生的地方。

---

## 一、计算图：把每一步运算都记下来

[Image 17]

要让 loss 修正参数，关键问题是：**每个参数对 loss 有多少影响？**

数学上叫"loss 对参数的偏导数"，工程上叫**梯度**。

得到梯度的算法叫**反向传播**。它听起来神秘，其实只是高中数学里的链式法则——一步步把"我对你的影响系数"算回去。

要做这件事，框架需要在前向传播的过程中，把每一步运算都**记下来**——记成一张图，叫**计算图**。

[Image 18]

microgpt 用一个叫 `Value` 的类来当图上的节点。每个 `Value` 记录三件事：

- **data**：自己当前的数值
- **_children**：自己是从哪几个 Value 算出来的（父节点）
- **_local_grads**：每个 child 对自己的"局部影响系数"

举两个例子：
- 加法 `a + b`：children 是 a 和 b，每个的局部梯度都是 1。
- 乘法 `a * b`：children 是 a 和 b，a 的局部梯度是 b.data，b 的局部梯度是 a.data。

每做一次运算，就生成一个新的 `Value`，并指回它的"父母"。整个前向过程跑下来，就织起一张从输入和参数、一路到 loss 的有向图。

这张图保存了完整的计算历史。loss 这个节点，可以沿着图反向找到所有影响过它的参数。

---

## 二、反向传播：梯度从 loss 流回所有参数

[Image 19]

到了 loss，我们调用 `loss.backward()`。

它做两件事：

第一，**拓扑排序**。从 loss 出发，按依赖关系把图里所有节点排好——保证算梯度的时候，每个节点的"下游"都已经算完了。

第二，**从 loss 开始往回灌梯度**。loss 自己对自己的梯度是 1。然后对每个节点 v，按局部梯度把 v 当前的梯度分发给它的 children：

> child.grad += local_grad × v.grad

`+=` 而不是 `=` 是因为一个 child 可能被多个 parent 用过，影响要**累加**。

这一段实现就十几行 Python，但它就是反向传播的全部。

每个参数最终积累到的 `grad`，就是 loss 对它的偏导数。这个数字告诉我们：**这个参数往哪个方向调整一点点，loss 就会变小一点点。**

---

## 三、Adam：更聪明的更新规则

[Image 20]

有了梯度，我们就可以更新参数了。最简单的更新规则是梯度下降：

> 参数 = 参数 − 学习率 × 梯度

但 microgpt 用的是 **Adam**，一个比朴素梯度下降聪明得多的优化器。它额外维护两个东西：

- **m**：梯度的"惯性"（动量）。最近往哪个方向走，就有点惯性继续往那走，不会被某次抖动带偏。
- **v**：梯度的"波动幅度"。波动大的参数，步子放小一点；波动小的参数，可以走快一点——也就是**自适应学习率**。

直观地讲：方向看 m，步长看 v 自适应调整。比纯梯度下降更稳、收敛更快。今天几乎所有大模型训练用的都是 Adam 或它的变种。

[Image 21]

还有一个小但关键的技巧：**学习率衰减**。

代码里这一行——`lr_t = learning_rate * (1 - step / num_steps)`——让学习率从 0.01 线性降到 0。

道理是：训练初期模型很差，离最优还远，可以大胆走；训练后期已经接近最优，再大步就会冲过头，要小心调整。

## 四、训练循环：把所有步骤串起来

[Image 22]

现在所有零件都齐了。把它们串起来，就是 microgpt 的训练循环：

每一步——
1. 取一个名字
2. 加 BOS、转 token id
3. 跑前向传播，算 loss
4. 调 `loss.backward()`，算梯度
5. 用 Adam 更新所有参数
6. 把所有参数的梯度清零，准备下一步

microgpt 跑了 **1000 步**。每步处理一个名字。打印出来的 loss 慢慢从 3 多降到 2 左右——模型确实在学到东西。

3 是什么概念？随机猜的 loss 大约是 ln(27) ≈ 3.3。降到 2 左右，说明模型已经从"瞎猜"走出来很远了。

整个训练过程，没有任何深奥的东西。就是这五步，重复 1000 遍。

## 五、推理：让模型生成新名字

[Image 23]

训练完，怎么用？

我们从 BOS 开始，让模型预测下一个字符。它输出 27 个 logits，过一次 softmax 变成概率分布。

然后——注意——我们**不是**直接取概率最高的字符，而是做**带温度的采样**。

代码里：`probs = softmax([l / temperature for l in logits])`，然后 `random.choices(...)` 按概率随机抽一个。

**temperature**（温度）是一个调节"创造性"的参数：

- 温度低（接近 0），分布很尖锐，模型几乎只选最高概率的字符——结果保守、重复。
- 温度高（接近 1 或更高），分布平滑，模型敢选低概率的字符——结果多样，但也容易出错。

ChatGPT 那个"重新生成"按钮，就是因为带了温度采样，每次结果都会不一样。

采到一个字符就接到序列后面，然后预测下一个，再采。一直到模型生成出 BOS——它在告诉你"我说完了"——这时候停下，输出一个完整的名字。

如果一直没采到 BOS，那就到 `block_size` 的上限（16 字符）强制停下。

## 六、从 microgpt 到 GPT-4

[Image 24]

讲完了。我们回头看一下。

microgpt 训练了一个生成英文名字的小模型，用 1000 步、4000 多个参数，跑在一台普通笔记本上。

GPT-4 训练了一个能写代码、答题、聊天的大模型，参数估计在万亿级别，用上万张 GPU 跑了几个月。

它们之间的差别在哪里？

- **数据更多**：互联网级别的文本，不是三万个名字
- **模型更深**：几十层，不是一层
- **分词更聪明**：BPE 子词，不是字符级
- **训练更久**：海量算力跑数月
- **训练完之后还要 SFT 和 RLHF**：才能学会"按人类指令回答问题"

但**核心机制——下一个字符预测、注意力、反向传播、Adam 优化、采样生成**——和你刚刚看到的 200 行 Python 代码一模一样。

如果你能看懂 microgpt.py，你就拿到了 LLM 训练的全部底牌。剩下的，是工程问题。
